# 产品指标体系学习与常用衡量表格笔记
---
## 一、 基础概念
### 1. 什么是产品指标体系？
* **定义**：一套结构化的、可量化的数值集合，用于描述、诊断、预测和驱动业务健康与增长。
* **核心目的**：统一度量衡 → 减少沟通成本 → 支持决策 → 指导优化迭代
* **常见分类**（由浅到深）：
  - 描述现状（发生了什么）
  - 拆解原因（为什么发生）
  - 预测趋势（接下来会怎样）
  - 驱动行动（我们该做什么）

#### 对比知识点：
| 产品类型       | 最核心关注指标（Top3）               | 典型关注周期    |
|:---------------|:-------------------------------------|-----------------|
| **内容/社交**  | DAU、时长、互动率                   | 日/周           |
| **电商/交易**  | GMV、转化率、客单价                 | 日/周/月        |
| **工具/效率**  | 日活、使用频次、核心功能完成功率     | 日/周           |
| **游戏**       | DAU、ARPPU、留存率、付费率           | 日/周/月        |

### 2. 指标层级（北极星指标 → 一级指标 → 二级指标）
- **北极星指标**（1个）：团队长期对齐的终极衡量目标
- **一级指标**（3–8个）：直接影响北极星的关键拆解维度
- **二级/三级指标**：诊断原因、定位问题用的明细指标

## 二、 常用核心指标速查表
### 1. 用户类指标
| 指标中文名       | 英文缩写   | 计算公式                          | 作用 / 关注点                        | 常见目标方向          |
|------------------|------------|-----------------------------------|--------------------------------------|-----------------------|
| 日活跃用户数     | DAU        | 当天去重登录/使用用户数           | 衡量产品整体活跃规模                 | ↑ 持续增长            |
| 月活跃用户数     | MAU        | 自然月内去重用户数                | 衡量用户基数与覆盖广度               | ↑ 长期健康            |
| 新增用户数       | DNU / 新增 | 当天首次使用用户数                | 增长引擎最直接体现                   | ↑ 拉新效果            |
| 次日留存率       | Day1 Retention | 新增用户第2天回访比例             | 产品是否真正解决用户问题（最重要留存指标） | >40%（消费）>25%（工具） |
| 7日留存率        | Day7       | 新增用户第8天回访比例             | 中期粘性与价值感知                   | >10–15%               |
| 人均使用时长     | Avg Duration | 总使用时长 / DAU                  | 内容/沉浸型产品的核心健康度         | ↑                     |
| 使用频次         | Frequency  | DAU / MAU                         | 用户粘性（越高越好）                 | >0.2–0.3（工具类更高）|

### 3. 商业/收入类指标
| 指标中文名       | 英文缩写   | 计算公式                          | 作用 / 关注点                        | 关注方向              |
|------------------|------------|-----------------------------------|--------------------------------------|-----------------------|
| 交易总额         | GMV        | 订单总金额（不扣退款）            | 电商/交易平台最重要规模指标          | ↑                     |
| 实收金额         | 收入       | GMV - 退款 - 渠道费等             | 实际到手收入                         | ↑                     |
| 客单价           | AOV / ARPU | 收入 ÷ 订单数 / 付费用户数        | 用户付费能力与结构                   | ↑（但不能牺牲转化）   |
| 付费率           | Paying Rate| 付费用户数 ÷ DAU/MAU              | 变现渗透率                           | 游戏5–15%，工具1–8%  |
| ARPPU            | ARPPU      | 收入 ÷ 付费用户数                 | 付费用户平均贡献                     | ↑                     |
| LTV              | Lifetime Value | 用户生命周期内贡献总收入          | 衡量用户终身价值                     | > CAC 3–5倍为健康     |

### 4. 健康/质量类指标（常被忽略但非常重要）
| 指标中文名           | 计算公式 / 定义                   | 关注点                               | 报警红线参考          |
|----------------------|-----------------------------------|--------------------------------------|-----------------------|
| 崩溃率               | 崩溃次数 / 启动次数               | App稳定性最直接指标                  | <0.5–1‰               |
| ANR/卡顿率           | ANR/卡顿次数 / 使用次数           | Android体验痛点                      | <1–2%                 |
| 加载成功率           | 成功加载次数 / 尝试次数           | 影响留存的第一体验关                 | >98–99%               |
| 核心功能完成功率     | 完成人数 / 进入人数               | 验证功能是否真正被使用               | >60–80%（视功能）     |
| NPS / 满意度         | 问卷净推荐值                      | 用户主观感知                         | >30为及格，>50优秀    |

## 三、 快速搭建指标看板建议（北极星模型）
推荐分层结构（由上到下）：

1. **顶层看板**（高管/周会用）  
   北极星指标 + 3–5个一级拆解指标 + 趋势折线 + 同比/环比

2. **部门/模块看板**（日/周运营用）  
   漏斗图 + 留存曲线 + 核心功能使用率 + 异常报警卡片

3. **分析诊断看板**（问题定位用）  
   多维度拆解（渠道、新老、地域、版本、设备…） + TopN 异常分析

常用可视化搭配建议：
- **核心数值** → 大数字卡片（带同比/环比颜色）
- **趋势** → 折线图 / 面积图
- **结构对比** → 柱状图 / 堆积图 / 饼图
- **转化路径** → 漏斗图
- **分布** → 直方图 / 箱型图
- **明细** → 透视表 / 表格

## 四、 学习心得与易错点
> ### 💡 经验总结
> 1. **先定北极星，再拆指标** —— 没有北极星的指标体系就是一盘散沙。
> 2. **留存 > 一切** —— 新用户次日留存是检验产品是否真正“有用”的最硬指标。
> 3. **口径必须统一** —— DAU到底算“启动即活”还是“至少停留5秒”？团队内必须对齐。
> 4. **不要唯数据论** —— 数据告诉你“什么发生了”，但很少直接告诉你“为什么”和“该怎么办”。
> 5. **关注“率”比“量”更重要** —— 规模上来后，各种率（留存、转化、付费、崩溃率）才是决定天花板的因素。
